## Namespaces

## 1. Kubernetes Namespaces

Namespaces are a native way to divide a single Kubernetes cluster into multiple *virtual clusters*

1. Kubernetes Namespaces are not the same as Linux kernel namespaces. *Kernel* *namespaces* divide operating systems into virtual operating systems called **containers**. Kubernetes Namespaces divide Kubernetes clusters into virtual clusters
2. Namespaces partition a Kubernetes cluster and are designed as an easy way to apply *quotas* and *policies* to groups of objects. They’re not designed for strong workload isolation

如果在K8S中创建对象的时候没有指定Namespace 的话， 系统会自动部署在 *default* Namespace下面

## 2.  Commands

```bash
# 查看名称空间 ns or namespace
kubectl get ns

# 查看ns 详情
kubectl describe namespace default

# 查看指定namespace下的svc 查看所有的话 就是 --all-namespaces
kubectl get svc -n kube-system

#查看所有的namespaces 下的svc
kubectl get svc --all-namespaces

# 删除namespace
kubectl delete ns xxx

# 切换当前的名称空间
 kubectl config set-context --current --namespace xxx
```

### 2.1 创建namespace

1. 直接command创建

```bash
 kubectl create ns hydra
```

2. 通过yaml文件创建

```yaml
kind: Namespace
apiVersion: v1
metadata:
  name: shield
  labels:
    env: marvel
```

```bash
kubectl apply -f shield-ns.yml
```
